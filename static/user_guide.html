  • Email Integration

System Control Commands

Control your computer using natural language commands. Important: Always specify your operating system (e.g., 'macos', 'windows', or 'linux') in system-related requests for better accuracy:

# Open Applications (macOS examples)
"Open Chrome and navigate to github.com macos"
"Launch Visual Studio Code macos"
"Open Terminal and run python script.py macos"

# Close Applications
"Close all Chrome windows macos"
"Quit Spotify macos"
"Close all inactive applications macos"

# System Operations
"Check available disk space macos"
"Show CPU usage macos"
"List all running processes macos"
"Create a new folder called 'Projects' on the desktop macos"
Always include your operating system (macos, windows, or linux) at the end of system-related requests. For example: "Close Chrome, open VS Code, and create a new file called test.py macos"

Direct Terminal Commands

You can execute terminal commands directly by entering them. Here are some useful macOS terminal commands:

# File and Directory Operations
ls -la                     # List all files with details
pwd                        # Show current directory
cd ~/Documents            # Change to Documents directory
mkdir -p folder/subfolder # Create nested directories
rm -rf folder            # Remove folder and contents
cp -R source dest        # Copy folder recursively
mv file.txt newname.txt  # Rename/move file

# System Information
top                      # Show running processes
df -h                    # Show disk space usage
ps aux | grep "Chrome"   # Find Chrome processes
system_profiler SPHardwareDataType  # Show hardware info
sysctl -n machdep.cpu.brand_string  # Show CPU info

# Network
ping google.com         # Test network connection
netstat -an | grep LISTEN  # Show listening ports
ifconfig                # Show network interfaces
curl ifconfig.me        # Show public IP address

# Process Management
killall "Google Chrome" # Close all Chrome windows
pkill -f "Firefox"      # Kill Firefox processes
launchctl list         # List running services

# System Maintenance
sudo purge             # Clear memory cache
softwareupdate -l      # List software updates
caffeinate            # Prevent system sleep
pmset -g              # Show power settings
You can combine terminal commands with natural language. For example: "Execute ls -la and then show me the disk space usage macos" or simply enter the command directly: "df -h"
For potentially destructive commands (like rm -rf), the system will ask for confirmation before executing them to prevent accidental data loss.

Email Integration Setup

The AI Workstation uses nmail for email functionality. Follow these steps to set up email integration:

1. Install nmail

# Install using Homebrew (macOS)
brew install nmail

2. Configure Gmail Integration

To use Gmail with the AI Workstation, you'll need to:

  1. Enable 2-Factor Authentication in your Google Account:
    • Go to your Google Account settings
    • Navigate to Security â†’ 2-Step Verification
    • Follow the prompts to enable 2FA
  2. Generate an App Password:
    • Go to your Google Account settings
    • Navigate to Security â†’ 2-Step Verification
    • Scroll to "App passwords" at the bottom
    • Select "Mail" as the app and your device type
    • Copy the 16-character password generated

3. Configure nmail

# Run the Gmail setup wizard
nmail -s gmail

# When prompted, enter:
# - Your Gmail address
# - The App Password you generated (NOT your regular Gmail password)
# - Your display name
The configuration files will be created in ~/.config/nmail/. The system will automatically use these for email operations.

4. Verify Configuration

The setup creates three important files in ~/.config/nmail/:

# main.conf - Primary configuration
address=your.email@gmail.com
name=Your Name
smtp_host=smtp.gmail.com
smtp_port=587
user=your.email@gmail.com

# auth.conf - Authentication details
# ui.conf - Interface preferences (optional)

5. Testing the Setup

After configuration, test the email integration with these commands:

# Simple email test
"Compose a test email to myself about email integration macos"

# Check if mail client opens correctly
"Open mail client macos"
If you encounter any issues:
  • Verify that your App Password was entered correctly
  • Check that 2FA is properly enabled on your Google Account
  • Ensure the configuration files exist in ~/.config/nmail/
  • Try regenerating a new App Password if needed

Using Email Integration

Once configured, use natural language to compose and send emails through your system's mail client. Remember to specify your operating system for mail client operations:

# Email Operations
"Compose an email to john@example.com about the project update macos"
"Open mail client and start a new message macos"
"Draft a professional email about scheduling a meeting macos"
The system will help structure your email content and can open your default mail client, but direct email checking and meeting invitations are handled through your mail client. Always include 'macos', 'windows', or 'linux' in system-related requests.
